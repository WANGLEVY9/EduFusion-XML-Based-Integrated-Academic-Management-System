## 1. 系统概述

EduFusion 是一个跨学院教务集成系统，模拟了三所独立学院（A 学院、B 学院、C 学院）的教学管理场景。系统核心能力：

- **课程共享**：各学院可将本校课程设为共享，供其他学院学生查看和选修
- **跨院选课**：学生可以选修其他学院的共享课程，选课记录自动写入课程所属学院
- **统一退选**：支持退选本院课程和跨院课程
- **全局统计**：聚合三个学院的数据，生成统计报表

### 三个学院

| 学院 | 账号前缀 | 学生范围 | 管理员账号 |
|------|---------|---------|-----------|
| **A 学院** | A | A001 ~ A060 | adminA |
| **B 学院** | B | B001 ~ B060 | adminB |
| **C 学院** | C | C001 ~ C060 | adminC |

> 每个学院使用不同的数据库系统（A：SQL Server，B：Oracle，C：MySQL），但对用户完全透明，操作方式完全一致。

## 2. 登录系统

### 启动客户端

分别启动三个学院的客户端程序，每个学院一个独立窗口（可同时运行）。

### 登录界面

登录窗口包含以下输入项：

| 字段 | 说明 | 示例 |
|------|------|------|
| **用户名** | 学生编号或管理员账号 | `A001` 或 `adminA` |
| **密码** | 登录密码 | 学生默认 `123456`，管理员默认 `admin123` |
| **身份** | 下拉选择"学生"或"管理员" | — |
| **学院** | 固定显示当前客户端所属学院（只读） | A / B / C |

### 操作步骤

1. 在"用户名"框输入账号
2. 在"密码"框输入密码
3. 在"身份"下拉框选择"学生"或"管理员"
4. 点击 **登录** 按钮

### 注意事项

- 用户名区分大小写，请准确输入
- 登录失败会弹出提示"用户名或密码错误"
- 同一学院的学生和管理员使用同一登录界面；仅管理员账号可登录管理员端

## 4. 课程查询

查看当前学院的全部课程列表。

### 操作步骤

1. 直接点击 **课程查询** 按钮
2. 表格将显示本院所有课程
3. 文本输出区显示课程数量和详情

### 表格列说明

| 列名 | 说明 |
|------|------|
| 课程号 | 课程唯一编号（如 A101、B205） |
| 课程名 | 课程名称 |
| 学分 | 该课程的学分数 |
| 教师 | 授课教师 |
| 地点 | 上课地点 |
| 学院 | 课程所属学院 |
| 共享 | 是否跨院共享（"是" / "否"） |

> 注意：课程查询仅显示**本院**课程。要查看其他学院的课程，请使用"共享课程选课"功能（课程编号留空时点击）。

## 5. 共享课程选课

选修其他学院开放的共享课程。此操作为**两步流程**。

### 前置条件

- 当前学院和课程所属学院**不同**
- 目标课程须标记为"共享"
- 尚未选修该课程

### 操作步骤

**第一步：加载可选共享课程**

1. 确保**课程编号**输入框**为空**
2. 点击 **共享课程选课** 按钮
3. 表格将显示其他学院开放的共享课程列表
4. 系统弹出提示"已加载可选共享课程"

**第二步：确认选课**

1. 在表格中点击目标课程所在行 → 课程编号自动填充到输入框
2. 再次点击 **共享课程选课** 按钮
3. 系统弹出确认对话框，显示学生编号和课程编号
4. 点击 **是** 确认选课

### 操作结果

- 选课成功：文本区显示"选课成功"，系统自动刷新"我的选课"
- 选课失败（如重复选课）：文本区显示失败原因

### 界面交互流程

```
点击【共享课程选课】（课程编号为空）
    → 加载其他学院共享课程列表到表格
    → 弹窗提示：已加载可选共享课程
    → 点击表格行 → 课程编号自动填充
    → 再次点击【共享课程选课】
    → 确认对话框：选课？
    → 确认 → 选课完成，自动刷新我的选课
```

## 6. 我的选课

查看当前学生已选的所有课程（包含本院课程和跨院选修的课程）。

### 操作步骤

1. 确保**学生编号**输入框已填写（默认自动填充当前登录账号）
2. 点击 **我的选课** 按钮
3. 表格显示已选课程列表

### 说明

- 输出区会显示已选课程数量和详细列表
- 如果要查看其他学生的选课情况，可手动修改"学生编号"后点击查询
- 此功能不区分本院和跨院课程，显示全部有效选课记录

---

## 7. 退课

退选已选的课程。

### 操作步骤

1. 先点击 **我的选课** 查看已选课程
2. 在表格中点击要退选的课程行 → 课程编号自动填充
3. 点击 **退课** 按钮
4. 系统弹出确认对话框
5. 点击 **是** 确认退课

### 操作结果

- 退课成功：文本区显示"退课成功"，系统自动刷新"我的选课"
- 退课失败：文本区显示失败原因

### 说明

- 退课不区分本院课程还是跨院课程，操作方式一致
- 退课不可撤销，如需重新选课需再次执行选课操作

## 8. 统计报表

查看系统的全局统计数据。

### 操作步骤

1. 点击 **统计报表** 按钮
2. 文本输出区显示统计信息，表格显示热门课程排行

### 统计内容

| 统计项 | 说明 |
|--------|------|
| **总学生数** | 三个学院的学生总数 |
| **总课程数** | 三个学院的课程总数 |
| **总选课数** | 全部选课记录总数 |
| **共享课程数** | 标记为共享的课程总数 |
| **热门课程 TOP** | 按选课人数排序的热门课程排行 |

### 热门课程排行

热门课程列表会显示在表格中，包含课程编号、课程名、所属学院和选课人数。

## 9. 表格交互

所有课程列表（课程查询、共享课程选课、我的选课、统计报表）都使用统一的表格控件，支持以下交互功能。

### 9.1 分页

当查询结果过多时，可使用分页功能浏览：

| 操作 | 方式 |
|------|------|
| **切换每页条数** | 在"每页"下拉框选择 5 / 10 / 20 / 50 条 |
| **上一页** | 点击 **上一页** 按钮（当前页为第一页时禁用） |
| **下一页** | 点击 **下一页** 按钮（当前页为最后一页时禁用） |
| **查看页码** | 顶部显示"第 X / Y 页（共 Z 条）" |

### 9.2 关键字筛选

在"筛选"输入框中输入关键字，表格将实时过滤显示匹配的行：

- 输入即过滤，无需点击按钮
- 不区分大小写
- 匹配范围：所有列（课程号、课程名、教师、学院等）
- 筛选基于当前查询结果，不会重新请求后端
- 清空筛选框即可恢复显示全部数据

**示例**：输入"数学" → 表格仅显示课程名中包含"数学"的课程；输入"A" → 显示所有含"A"的记录。

### 9.3 导出 CSV

将当前筛选结果导出为 CSV 文件：

1. 点击 **导出当前筛选** 按钮
2. 在文件选择对话框中选择保存位置和文件名
3. 默认文件名为 `courses-{学院}-{日期时间}.csv`
4. 导出成功后会弹出提示

**导出说明**：
- 仅导出当前筛选后的数据（如未设置筛选，则导出全部查询结果）
- CSV 文件可用 Excel、WPS 或文本编辑器打开
- 如果表格为空（未查询或筛选无结果），则会提示"当前无可导出的行"

### 9.4 点击填充课程编号

点击表格中的任意行，该行第一列（课程号）会自动填充到顶部的"课程编号"输入框：

- 填充后可直接点击 **共享课程选课** 或 **退课** 执行对应操作
- 文本输出区会提示"已选择课程：{课程编号}，可直接执行选课或退课。"

## 10. 常见问题

### 10.1 登录失败

**现象**：点击登录后弹出"用户名或密码错误"。

**检查**：
- 确认用户名和密码大小写正确
- 确认选择了正确的"身份"（学生 / 管理员）
- 确认启动的是对应学院的客户端（如使用 A 学院客户端登录 A 学院账号）
- 确认集成服务器（:8080）已启动

### 10.2 查询无数据

**现象**：点击"课程查询"后表格为空。

**检查**：
- 确认数据库已初始化（包含学生、课程和选课数据）
- 检查集成服务是否正常运行
- 尝试点击"统计报表"看是否能返回数据

### 10.3 选课失败

**现象**：确认选课后提示失败。

**常见原因**：
- 已选修过该课程（不可重复选课）
- 目标课程不是共享课程
- 选本院课程（跨院选课功能仅支持选修**其他学院**的课程）
- 课程编号输入错误

### 10.4 退课失败

**现象**：确认退课后提示失败。

**常见原因**：
- 课程编号输入错误
- 该选课记录不存在
- 学生编号与选课记录不匹配

### 10.5 页面数据显示不全

**建议**：
- 调整"每页"下拉框的条数（选择 20 或 50）
- 使用"筛选"功能缩小搜索范围
- 重新点击对应的查询按钮刷新数据
